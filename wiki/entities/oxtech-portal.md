# Oxtech Portal

Центральный компонент [[oxtech-platform]]: Rails 8.1 + React 19. Портал — это одновременно
конструктор агентов, сервер данных, реестр и панель управления. Всё, что агенты знают о себе
и о мире, приходит отсюда.

## Роли

| Роль | Что делает |
|---|---|
| Конструктор агентов | визуальная настройка: роль, промпт, модель, набор MCP-инструментов, каналы, права. Новый агент — конфигурация, не разработка |
| MCP-сервер | 67 инструментов доступа к корпоративным данным ([[mcp]]) |
| Реестр и источник конфигурации | агент на каждом старте тянет конфиг из Portal; здесь же матрица «кто кого может вызвать» по A2A |
| Админка / наблюдаемость | метрики в реальном времени, траектории, граф вызовов ([[observability]]) |
| Веб-чат | канал общения с агентами прямо в интерфейсе |
| Слой данных | пользователи, проекты, услуги, факты, почта, финансы, календарь, база знаний с векторным поиском |

## Источник правды для агентов

Ключевой архитектурный принцип: конфигурация агента (системный промпт, модель, инструменты,
разрешённые агенты, настройки) живёт в Portal, а не в коде агента. Агент стартует с тремя
переменными окружения и запрашивает остальное по API. Отсюда:

- изменение поведения агента — правка в админке, а не пересборка;
- платформенные настройки (например, тумблер [[proactive-messenger-mode]]) применяются
  централизованно;
- без Portal агент не поднимется (fail-fast), агентов со старым конфигом не остаётся.

## Управление жизненным циклом

Массовые операции над системой агентов (Deploy / Restart / Rebuild / Stop All) с учётом
зависимостей и порядка запуска по тирам. Фактический деплой выполняет [[oxtech-manager]];
Portal ставит задачи и отображает статусы.

## Аутентификация

Bearer-токен (персональный, агентский, системный). Для внешних ассистентов — OAuth 2.1 через
конвертер. Разграничение инструментов на агента — белый список, проверяется на сервере ([[mcp]]).

На платформе работают прикладные продукты: [[oxdocs]] (документы), [[keylink]] (закупки) —
оба используют данные через MCP Portal.
