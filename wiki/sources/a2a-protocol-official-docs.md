# Официальные материалы протокола A2A

Конспект снапшота github.com/a2aproject/A2A и a2a-protocol.org на 2026-07-20.
Полный текст — [[raw/2026-07-20_a2a-protocol-official-docs.md]].

## Что зафиксировано

| Факт | Значение |
|---|---|
| Управление | Linux Foundation, вклад Google, Apache 2.0 |
| Версия спецификации | **v1.0.1**, выпущена 28.05.2026, всего 11 релизов |
| Транспорт | JSON-RPC 2.0 поверх HTTP(S) + SSE |
| SDK | 6 языков: Python, Go, JavaScript, Java, .NET, Rust |
| Обнаружение | well-known URI, курируемые реестры, прямая конфигурация |

## Три принципа проектирования

1. Simplicity — HTTP, JSON-RPC, SSE вместо проприетарных стандартов.
2. Enterprise-grade — production, безопасность, комплаенс.
3. Asynchronous-first — длительные задачи и разрывы соединения нативно.

## Четыре заявленных преимущества

Совместимость между фреймворками, безопасность через непрозрачность (агент не раскрывает
внутреннюю логику, память и проприетарные инструменты), корпоративная готовность
(аутентификация, авторизация, мониторинг), мультимодальность.

## Важное для [[dit-agent-factory-concept]]

Спецификация описывает **курируемые реестры** как штатную стратегию обнаружения для
корпоративных сред, но **не стандартизует API реестра**. Это прямо соответствует идее
«комнат-рынков навыков» и означает: реализация реестра — легитимное расширение поверх
стандарта, а не отклонение от него.

## Не установлено

Список компаний-адоптеров в репозитории не приведён — только программа партнёрства для
клиентов Google Cloud. Утверждения о числе поддерживающих вендоров требуют отдельной
проверки перед использованием в публичных материалах.
