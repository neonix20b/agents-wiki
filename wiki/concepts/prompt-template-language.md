# Язык шаблонов промптов (JSON-DSL)

Идея Д. Смолева (2026-07-20) для концепции ДИТ: промпт-инжиниринг на базе JSON с
постановкой на создание **специализированного языка шаблонов** вместо монолитных
текстовых промптов.

## Суть

Роль агента описывается не сплошным текстом, а структурой с переменными:

```json
{
  "rules": "Всегда выбирай красное",
  "prompt": "Система должна следовать следующим правилам: {rules}"
}
```

Переменные — от констант до целых правил и повторяющихся наборов инструкций —
**переиспользуются** между шаблонами и агентами.

## Следствия для платформы

- **Единая правка** — общее правило (тон обращения, запреты, формат протокола)
  меняется в одном месте и применяется ко всем агентам, использующим переменную.
- **Композиция ролей**: специальность = базовый шаблон + отраслевые наборы правил +
  локальные параметры ведомства.
- **Версионирование и диффы**: структура диффится и версионируется как код; правка
  регламента — это reviewable-изменение, а не переписывание «простыни».
- **Валидация**: схема позволяет проверять заполненность переменных до выпуска
  агента (связь с конвейером аттестации — шаблон тестируется эталонными заданиями).
- **Разделение труда**: методологи ведомств правят правила-значения, инженеры —
  структуру шаблонов.

## Открытые вопросы проработки

Синтаксис ссылок ({var} vs $ref), области видимости (платформа → комната → агент),
наследование/переопределение, escaping, каталог переменных в реестре, миграция
существующих промптов. Прецеденты для изучения: Jinja-подобные шаблонизаторы,
Prompty (Microsoft), DSPy, POML.
